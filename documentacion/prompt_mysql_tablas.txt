# Prompt para creación de Tablas en MySQL Server

Actúa como un experto en bases de datos y MySQL. Genera el script SQL para crear la estructura de base de datos de una aplicación de gestión de propiedades turísticas (tipo Airbnb).

Basado en los siguientes requerimientos y estructura preliminar, por favor genera el código DDL (Data Definition Language) COMPLETO para MySQL.

**Instrucciones específicas:**
1.  Usa sintaxis compatible con **MySQL Server**.
2.  Define claves primarias (`AUTO_INCREMENT`) y claves foráneas correctamente para mantener la integridad referencial (`ON DELETE CASCADE` donde tenga sentido).
3.  Utiliza tipos de datos apropiados:
    *   `DECIMAL(10,2)` para importes monetarios.
    *   `DECIMAL(10,8)` para latitud/longitud.
    *   `VARCHAR` con longitudes adecuadas para textos cortos.
    *   `TEXT` para descripciones largas o JSON stringified.
    *   `DATETIME` o `TIMESTAMP` para marcas de tiempo.
    *   `BOOLEAN` o `TINYINT(1)` para booleanos.
4.  **Mejoras aplicadas sobre el diseño original:**
    *   Tabla `viviendas`: Se han añadido `latitud` y `longitud` para la geolocalización (Requisito RF-13).
    *   Tabla `huespedes`: Se han añadido campos necesarios para el parte de viajeros a la policía (`tipo_documento`, `fecha_expedicion_documento`, `sexo`).
    *   Tabla `transacciones`: Se ha vinculado con `id_categoria` en lugar de guardar el nombre de la categoría como texto, y opcionalmente con `id_reserva`.
    *   Tabla `reservas`: Se ha añadido `id_reserva_externo` para gestionar la sincronización con calendarios (RF-26).

---

## Estructura de Tablas Sugerida (SQL)

```sql
-- Creación de la base de datos (Opcional)
-- CREATE DATABASE gestion_propiedades;
-- USE gestion_propiedades;

-- 1. Tabla Usuarios
CREATE TABLE usuarios (
    id_usuario INT AUTO_INCREMENT PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL,
    apellidos VARCHAR(100) NOT NULL,
    email VARCHAR(150) NOT NULL UNIQUE,
    password_hash VARCHAR(255),
    telefono VARCHAR(20),
    google_id VARCHAR(255),
    fecha_alta TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    activo BOOLEAN DEFAULT TRUE,
    -- Campos para recuperación de contraseña (RF-05, RF-06)
    reset_password_token VARCHAR(255),
    reset_password_expires DATETIME
);

-- 2. Tabla Viviendas
CREATE TABLE viviendas (
    id_vivienda INT AUTO_INCREMENT PRIMARY KEY,
    id_usuario INT NOT NULL,
    direccion VARCHAR(255),
    ciudad VARCHAR(100),
    codigo_postal VARCHAR(20),
    barrio VARCHAR(100),
    tipo_vivienda VARCHAR(50), -- Apartamento, Casa, Villa, etc.
    num_banos INT,
    num_habitaciones INT,
    capacidad_maxima INT,
    superficie DECIMAL(10,2), -- Equivalente a 'm2_useful' del modelo
    
    -- Campos específicos para Modelos Predictivos (RF-58, RF-60)
    planta INT, -- 'floor' para el modelo
    ano_construccion INT, -- 'construct_date' para el modelo (calculo de edad)
    superficie_parcela DECIMAL(10,2), -- 'ground_size' para el modelo

    amenities TEXT, -- Se puede guardar como JSON string o lista separada por comas
    imagen_url TEXT,
    estado_legal VARCHAR(50),
    referencia_catastral VARCHAR(100),
    numero_registro_autonomico VARCHAR(100),
    fecha_registro DATE,
    activa BOOLEAN DEFAULT TRUE,
    latitud DECIMAL(10, 8), -- Necesario para Mapas (RF-13)
    longitud DECIMAL(11, 8), -- Necesario para Mapas (RF-13)

    -- Campos para Sincronización de Calendarios (RF-26)
    url_ical_airbnb TEXT,
    url_ical_booking TEXT,
    
    FOREIGN KEY (id_usuario) REFERENCES usuarios(id_usuario) ON DELETE CASCADE
);

-- 3. Tabla Reservas
CREATE TABLE reservas (
    id_reserva INT AUTO_INCREMENT PRIMARY KEY,
    id_vivienda INT NOT NULL,
    fecha_inicio DATE NOT NULL,
    fecha_fin DATE NOT NULL,
    num_huespedes INT,
    importe_total DECIMAL(10,2),
    plataforma VARCHAR(50), -- 'Airbnb', 'Booking', 'Propia', etc.
    id_reserva_externo VARCHAR(255), -- ID de la reserva en la plataforma externa (RF-26)
    estado VARCHAR(50), -- 'Confirmada', 'Pendiente', 'Cancelada', 'Bloqueada'
    checkin_realizado BOOLEAN DEFAULT FALSE,
    checkout_realizado BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (id_vivienda) REFERENCES viviendas(id_vivienda) ON DELETE CASCADE
);

-- 4. Tabla Huespedes
CREATE TABLE huespedes (
    id_huesped INT AUTO_INCREMENT PRIMARY KEY,
    nombre VARCHAR(100),
    apellidos VARCHAR(100),
    dni VARCHAR(50), -- Documento de Identidad
    tipo_documento VARCHAR(50), -- 'DNI', 'Pasaporte' (Necesario para RF-33 Policía)
    fecha_expedicion_documento DATE, -- (Necesario para RF-33 Policía)
    email VARCHAR(150),
    telefono VARCHAR(20),
    nacionalidad VARCHAR(50),
    sexo CHAR(1), -- 'M', 'F' (Necesario para RF-33 Policía)
    fecha_nacimiento DATE
);

-- 5. Tabla Intermedia Reserva_Huesped
CREATE TABLE reserva_huesped (
    id_reserva INT NOT NULL,
    id_huesped INT NOT NULL,
    PRIMARY KEY (id_reserva, id_huesped),
    FOREIGN KEY (id_reserva) REFERENCES reservas(id_reserva) ON DELETE CASCADE,
    FOREIGN KEY (id_huesped) REFERENCES huespedes(id_huesped) ON DELETE CASCADE
);

-- 6. Tabla Categorias Financieras
CREATE TABLE categorias_financieras (
    id_categoria INT AUTO_INCREMENT PRIMARY KEY,
    id_usuario INT NOT NULL,
    nombre VARCHAR(100),
    tipo ENUM('ingreso', 'gasto') NOT NULL,
    FOREIGN KEY (id_usuario) REFERENCES usuarios(id_usuario) ON DELETE CASCADE
);

-- 7. Tabla Transacciones (Ingresos y Gastos)
CREATE TABLE transacciones (
    id_transaccion INT AUTO_INCREMENT PRIMARY KEY,
    id_vivienda INT NOT NULL,
    id_reserva INT, -- Opcional: para vincular un ingreso a una reserva específica
    id_categoria INT, -- Vinculación con categoría personalizada
    tipo ENUM('ingreso', 'gasto') NOT NULL,
    importe DECIMAL(10,2) NOT NULL,
    fecha DATE NOT NULL,
    descripcion TEXT,
    FOREIGN KEY (id_vivienda) REFERENCES viviendas(id_vivienda) ON DELETE CASCADE,
    FOREIGN KEY (id_reserva) REFERENCES reservas(id_reserva) ON DELETE SET NULL,
    FOREIGN KEY (id_categoria) REFERENCES categorias_financieras(id_categoria) ON DELETE SET NULL
);

-- 8. Tabla Tareas
CREATE TABLE tareas (
    id_tarea INT AUTO_INCREMENT PRIMARY KEY,
    id_vivienda INT NOT NULL,
    descripcion TEXT,
    fecha_limite DATE,
    estado VARCHAR(50) DEFAULT 'Pendiente', -- 'Pendiente', 'Completada'
    vinculada_calendario BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (id_vivienda) REFERENCES viviendas(id_vivienda) ON DELETE CASCADE
);

-- 9. Tabla Incidencias
CREATE TABLE incidencias (
    id_incidencia INT AUTO_INCREMENT PRIMARY KEY,
    id_vivienda INT NOT NULL,
    concepto VARCHAR(150),
    descripcion TEXT,
    fecha_reporte DATE,
    resuelta BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (id_vivienda) REFERENCES viviendas(id_vivienda) ON DELETE CASCADE
);

-- 10. Tabla Documentos Vivienda
CREATE TABLE documentos_vivienda (
    id_documento INT AUTO_INCREMENT PRIMARY KEY,
    id_vivienda INT NOT NULL,
    tipo_documento VARCHAR(100), -- 'Escritura', 'Seguro', 'Licencia'
    url_documento TEXT,
    fecha_subida TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (id_vivienda) REFERENCES viviendas(id_vivienda) ON DELETE CASCADE
);
```
